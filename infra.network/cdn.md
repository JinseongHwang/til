## 언제 CDN이 필요한가?

1. 오리진 서버가 단일 지역에 있지만, 글로벌 고객에게 서비스 해야할 때. 오리진 서버에서 멀리 떨어진 고객이 성능 문제로 제품에 대한 나쁜 경험이 예상된다면 CDN을 도입해야 한다. (ex. 쇼핑몰 상품 데이터 캐싱)
2. 실시간 스트리밍을 해야할 때.

## CDN은 캐싱 용도 아닌가? 실시간 스트리밍에 도움이 되나?

CDN은 정적 콘텐츠 캐싱보다는 엣지 서버를 통한 지리적 분산이 핵심이다.
그치만! 라이브 스트리밍도 실제로는 작은 청크 단위로 캐싱된다.

스트리밍 방식에 HLS/DASH 등의 방식이 있는데,
- 라이브 방송을 -> 2~10초 세그먼트로 분할해서 -> 각 세그먼트를 엣지 서버로 캐싱한다.
- 각 세그먼트는 생성되면 불변이고, 캐시 가능한 상태로 생성한다.
  - segment_001.ts (2초 분량 생성 완료)
  - segment_002.ts (2초 분량 생성 완료)
  - segment_003.ts (2초 분량 생성 중,,,)

그리고 CDN 구축 시 오리진 서버와 엣지 서버가 데이터 주고 받을 때 QUIC or 전용 회선을 사용해서 안정적이고 빠르게 전송한다. 따라서 오리진 서버에서 엔드 유저에게 직접 데이터를 전달하는 것보다 효율적이다.
